# 技术架构文档

## 1. 技术栈概览

| 模块 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| **核心框架** | Vue 3 | 3.x | 采用 Composition API 组织逻辑 |
| **构建工具** | Vite | 4.x | 提供极速开发体验和打包 |
| **3D 引擎** | Babylon.js | 8.x | 负责场景渲染、材质、交互 |
| **UI 交互** | Vue 3 Reactive | - | 摄像机参数实时监控与显示 |
| **动画引擎** | Tween.js | - | 负责摄像机运镜、模型位移动画 |
| **几何计算** | Earcut | - | 用于 GeoJSON 多边形的三角剖分 |
| **工具库** | Lodash | - | 数据处理辅助 |

## 2. 系统架构

系统采用单页应用 (SPA) 架构，以 `App.vue` 为核心容器，通过 Hooks 拆分业务逻辑。

```mermaid
graph TD
    A[main.js] --> B[App.vue]
    B --> C[Map3d.js (工具类)]
    B --> D[Hooks (逻辑复用)]

    subgraph Core_Hooks
    D --> D1[useFileLoader]
    D --> D2[useConversionStandardData]
    D --> D3[useMapMarkedLightPillar]
    D --> D4[useCountry]
    D --> D5[useRisingParticles]
    end

    subgraph Rendering_Pipeline
    C --> E[Babylon Engine]
    E --> F[Scene]
    F --> G[MeshBuilder]
    F --> H[StandardMaterial]
    F --> I[GlowLayer]
    end
```

## 3. 核心模块详解

### 3.1 地图生成管线 (`App.vue` + `useConversionStandardData`)
1.  **数据加载**: `useFileLoader` 请求 GeoJSON（浙江省：`public/data/map/浙江省.geojson`）。
2.  **坐标标准化**: `useConversionStandardData` 将 Polygon 与 MultiPolygon 统一为多维数组结构。
3.  **网格构建**: `MeshBuilder.ExtrudePolygon` 根据坐标生成有厚度的 3D 模型。
4.  **UV 映射**: 自动计算每个顶点的 UV，确保整张纹理完整覆盖在拼合后的地图上。

### 3.2 渲染循环与动画
*   **渲染循环**: 依赖 `engine.runRenderLoop`。
*   **动画更新**: 在渲染循环中调用 `TWEEN.update()` 驱动所有缓动动画。
*   **自适应逻辑**: 利用 `scene.registerBeforeRender` 监听摄像机半径 (`camera.radius`)，实时调整光柱 (`lightPillars`) 的 `scaling` 属性。
    *   `radius < 8` 时：`scale = radius / 8`（最近不可小于 0.2）
    *   `radius > 8` 时：`scale = 8 / radius`（最远不可小于 0.3）

### 3.3 交互系统
*   **事件监听**: 使用 Babylon.js 的 `ActionManager` 监听网格的点击事件。
*   **状态管理**:
    *   `currentSelectedMesh`: 记录当前选中的网格。
    *   通过修改 `material` 引用（`highlightMat` vs `dimMat`）来切换视觉状态。
    *   通过 `TWEEN` 动画改变 `position.y` 实现地块升降（抬升 0.2 单位）。
    *   使用 `setEnabled()` 控制复合模型（如光柱）的显隐。

### 3.4 材质系统
*   **StandardMaterial**: 项目主要使用标准材质，配合 `disableLighting = true` 实现自发光效果。
*   **GlowLayer**: 用于实现辉光效果。
    *   *优化策略*: 将基础地图 Mesh 加入 `addExcludedMesh` 列表，防止全局模糊，仅让高亮区域和光效组件发光。

## 5. 扩展性设计
*   **数据解耦**: 地图形状完全由 GeoJSON 数据决定，更换数据即可渲染不同省份（当前为浙江省）。
*   **配置分离**: 关键视觉参数（颜色、高度、光照强度）集中在 `App.vue` 的 `initModel` 和 `initLight` 中，便于调整。
*   **摄像机校准**: 内置调试面板可实时查看摄像机参数，方便记录和还原最佳视角。

## 6. 关键数据流

```
User Click → ActionManager → 选中 Mesh
    → TWEEN 抬升 (y: 0 → 0.2)
    → 切换材质 (mat → highlightMat)
    → 隐藏其它光柱 (setEnabled(false))
    → TWEEN 运镜 (camera.target → mesh center)
    → 选中其它 Mesh → 复原前一个 + 选中新的
    → 再次点击同一 Mesh → 全部复原
```
