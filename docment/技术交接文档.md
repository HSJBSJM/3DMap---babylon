# 技术交接文档

## 1. 环境准备
### 1.1 依赖环境
*   **Node.js**: 推荐 v16.x 或 v18.x LTS 版本。
*   **包管理器**: npm 或 pnpm。

### 1.2 安装步骤
```bash
# 1. 克隆/下载项目代码
# 2. 进入项目目录
cd 3DMap-babylon

# 3. 安装依赖
npm install
```

### 1.3 启动开发服务
```bash
npm run dev
```
访问 `http://localhost:5173` (或控制台提示的端口)。

### 1.4 生产打包
```bash
npm run build
```
打包产物位于 `dist/` 目录，可部署至任何静态文件服务器 (Nginx, IIS 等)。

## 2. 关键配置与维护

### 2.1 样式调整 (`src/App.vue`)
所有核心视觉参数均位于 `initModel` 和 `initLight` 方法中：

| 参数项 | 变量名 | 当前值 | 作用 |
|--------|--------|--------|------|
| 底图基础色 | `mat.diffuseColor` | `#0E4040` | 地图在光照下的固有色 |
| 底图自发光 | `mat.emissiveColor` | `#2C8080` | 地图在暗处的亮度 |
| 高亮颜色 | `highlightMat.diffuseColor` | `#22FFFF` | 选中状态的颜色 |
| 暗化颜色 | `dimMat.diffuseColor` | `#051515` | 未选中状态的颜色 |
| 边界线高度 | `lineTop.position.y` | `0.03` | 白色轮廓线的悬浮高度 |
| 边界线高度 | `lineBottom.position.y` | `0.05` | 底部暗色轮廓线高度 |
| 光柱缩放因子 | `scaleFactor` | `0.45` | 光柱、标注、光圈整体大小 |
| 光柱高度倍数 | `heightScaleFactor` | `0.45` | 单根光柱高度 |

## 3. 常见问题 (FAQ)

**Q1: 地图看起来太暗/太亮怎么办？**
*   A: 调整 `App.vue` 中 `mat.emissiveColor` 的颜色值。越接近白色越亮，越接近黑色越暗。同时可调整 `initLight` 中的光照强度。

**Q2: 光柱为什么没有随着相机拉远而变大？**
*   A: 请检查 `App.vue` 中的 `scene.registerBeforeRender` 逻辑。目前设定是 `radius > 8` 时开始缩小，且有 `0.3` 的最小值限制；`radius < 8` 时也按比例缩小，最小 `0.2`。

**Q3: 点击地市后，标签文字为什么没有跟着浮起来？**
*   A: 目前设计是标签锚点固定在 `y=0`。如果需要标签随模型抬起，需要将标签的 `linkWithMesh` 目标绑定到可动的 `extruded` 网格上，而不是静态的 `anchor` 盒子。但为了整齐，目前保持固定不动。

**Q4: 浙江省的地图数据和之前的不同怎么办？**
*   A: 项目已从酒泉市（县级）切换为浙江省（地市级），数据结构不变。如果需切换其他省份，只需更换 `浙江省.geojson` 并调整摄像机参数。

## 4. 遗留事项
*   光柱的隐藏逻辑使用了 `setEnabled`，这是为了兼容 `TransformNode` 的层级结构，请勿随意改为 `isVisible`。
*   装饰层的透明度已强制设为 `1.0` (`alpha = 1.0`)，且禁用了光照影响 (`disableLighting = true`)，以保证最亮显示。
*   当前纹理 `gz-map.jpg` 是甘肃底图，后续需替换为浙江风格纹理以实现正确的视觉效果。
*   光柱、标注、光圈大小为原来的 30%（`scaleFactor: 0.45`），可在 `useMarkedLightPillar` 配置中统一调整。
